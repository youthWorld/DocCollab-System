# 需求规格说明书（SRS）

## 一、引言

### 1.1 目的

本文档旨在详细描述"多人文档协作系统（DocCollab）"的功能需求与非功能需求，为后续的设计、开发、测试提供依据。本文档遵循CMMI需求管理过程域的要求，确保需求的完整性、可追溯性和可验证性。

### 1.2 范围

DocCollab 是一款面向研究生课程小组项目的轻量级协作管理工具，以"版本容器"为核心概念，提供文档协作编辑、文件管理、版本追溯等功能。

### 1.3 定义与缩写

| 术语     | 说明                                                    |
| -------- | ------------------------------------------------------- |
| 版本容器 | 保存一次完整项目状态的快照单元，包含文档、代码、资源等  |
| SRS      | Software Requirements Specification，软件需求规格说明书 |
| OT       | Operational Transformation，操作变换算法                |

---

## 二、总体描述

### 2.1 产品视角

本系统为独立的Web应用程序，采用前端浏览器本地存储（LocalStorage/IndexedDB），无需后端服务器。用户通过浏览器直接访问，在同一浏览器环境下进行协作。

### 2.2 用户特征

- **主要用户**：研究生课程小组成员
- **技术水平**：具备基本的计算机操作能力
- **使用场景**：课程项目协作，包括文档编写、代码管理、资源整理

## 三、功能需求

### 3.1 v1.0 核心功能（必须有）

#### R1：版本容器管理

| 属性         | 内容                                                                                                 |
| ------------ | ---------------------------------------------------------------------------------------------------- |
| **编号**     | R1                                                                                                   |
| **功能名称** | 版本容器管理                                                                                         |
| **版本**     | v1.0                                                                                                 |
| **优先级**   | 高                                                                                                   |
| **描述**     | 用户可创建版本容器，每个容器代表一次完整的项目状态快照。容器内包含当前所有文档、代码文件和资源文件。 |
| **输入**     | 容器名称                                                                                             |
| **处理**     | 系统创建新的版本容器，将当前工作区的所有内容打包保存为新版本                                         |
| **输出**     | 新的版本容器，包含版本号、创建时间、包含的文件列表                                                   |
| **约束**     | 容器名称不可为空                                                                                     |

#### R2：文档协作编辑

| 属性         | 内容                                                                                             |
| ------------ | ------------------------------------------------------------------------------------------------ |
| **编号**     | R2                                                                                               |
| **功能名称** | 文档协作编辑                                                                                     |
| **版本**     | v1.0                                                                                             |
| **优先级**   | 高                                                                                               |
| **描述**     | 提供富文本编辑器，支持多人同时编辑同一篇文档。支持基本的文本格式化（加粗、斜体、标题、列表等）。 |
| **输入**     | 文档内容编辑操作                                                                                 |
| **处理**     | 通过富文本编辑器进行文档编辑，变更实时同步到其他协作用户                                         |
| **输出**     | 更新后的文档内容                                                                                 |
| **约束**     | 基于浏览器LocalStorage存储，协作限于同一浏览器环境                                               |

#### R3：资源上传下载

| 属性         | 内容                                                                                |
| ------------ | ----------------------------------------------------------------------------------- |
| **编号**     | R3                                                                                  |
| **功能名称** | 资源上传下载                                                                        |
| **版本**     | v1.0                                                                                |
| **优先级**   | 高                                                                                  |
| **描述**     | 支持代码文件、图片、视频等资源文件的上传和下载。上传的文件存储在浏览器IndexedDB中。 |
| **输入**     | 本地文件                                                                            |
| **处理**     | 将文件读取并存储到IndexedDB，关联到当前容器                                         |
| **输出**     | 上传成功的文件列表；下载时输出原始文件                                              |
| **约束**     | 受浏览器本地存储容量限制（通常数百MB），单个大文件建议不超过50MB                    |

### 3.2 v1.1 迭代功能

#### R4：修改标签与描述

| 属性         | 内容                                                                                                              |
| ------------ | ----------------------------------------------------------------------------------------------------------------- |
| **编号**     | R4                                                                                                                |
| **功能名称** | 修改标签与描述                                                                                                    |
| **版本**     | v1.1                                                                                                              |
| **优先级**   | 中                                                                                                                |
| **描述**     | 每次创建新版本容器时，用户必须选择修改标签（如：需求变更、Bug修复、功能新增、文档更新、代码重构）并填写修改描述。 |
| **输入**     | 修改标签、修改描述文本                                                                                            |
| **处理**     | 系统校验标签和描述非空，将标签和描述关联到新版本容器                                                              |
| **输出**     | 带有标签和描述的版本记录                                                                                          |
| **约束**     | 标签为必选项，描述为必填项（不少于5个字符）                                                                       |

#### R5：变更记录

| 属性         | 内容                                                                                         |
| ------------ | -------------------------------------------------------------------------------------------- |
| **编号**     | R5                                                                                           |
| **功能名称** | 变更记录查看                                                                                 |
| **版本**     | v1.1                                                                                         |
| **优先级**   | 中                                                                                           |
| **描述**     | 用户可查看当前项目容器内所有变更历史，包括每次变更的版本号、时间、标签、描述和变更内容概要。 |
| **输入**     | 选择查看的项目                                                                               |
| **处理**     | 系统从存储中读取该项目的所有版本容器记录，按时间倒序排列                                     |
| **输出**     | 变更记录列表                                                                                 |
| **约束**     | 无                                                                                           |

### 3.3 v1.2 迭代功能

#### R6：版本对比

| 属性         | 内容                                                                                                           |
| ------------ | -------------------------------------------------------------------------------------------------------------- |
| **编号**     | R6                                                                                                             |
| **功能名称** | 版本对比                                                                                                       |
| **版本**     | v1.2                                                                                                           |
| **优先级**   | 中                                                                                                             |
| **描述**     | 用户可选择两个版本容器进行对比，系统展示两个版本之间的差异，包括新增文件、删除文件、修改文件以及文档内容差异。 |
| **输入**     | 选择两个版本号                                                                                                 |
| **处理**     | 系统对比两个版本的文件列表和文档内容，计算差异                                                                 |
| **输出**     | 差异对比结果（新增/删除/修改清单及内容diff）                                                                   |
| **约束**     | 只能对比同一项目内的两个版本                                                                                   |

### 3.4 v2.0 完善功能

#### R7：容器分支

| 属性         | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| **编号**     | R7                                                         |
| **功能名称** | 容器分支                                                   |
| **版本**     | v2.0                                                       |
| **优先级**   | 低                                                         |
| **描述**     | 支持从任意历史版本容器创建分支，分支可独立演进，互不影响。 |
| **输入**     | 选择源版本、分支名称                                       |
| **处理**     | 以源版本的完整内容创建新分支，后续修改仅影响当前分支       |
| **输出**     | 新创建的分支                                               |
| **约束**     | 分支名称不可与已有分支重名                                 |

#### R8：批量导出

| 属性         | 内容                                                      |
| ------------ | --------------------------------------------------------- |
| **编号**     | R8                                                        |
| **功能名称** | 批量导出                                                  |
| **版本**     | v2.0                                                      |
| **优先级**   | 低                                                        |
| **描述**     | 支持一键导出指定版本容器内的所有文件，打包为ZIP文件下载。 |
| **输入**     | 选择要导出的版本                                          |
| **处理**     | 系统收集该版本的所有文件，打包为ZIP并触发下载             |
| **输出**     | ZIP压缩包                                                 |
| **约束**     | 大文件导出受浏览器内存限制                                |

#### R9：容器标签

| 属性         | 内容                                                                                               |
| ------------ | -------------------------------------------------------------------------------------------------- |
| **编号**     | R9                                                                                                 |
| **功能名称** | 容器标签                                                                                           |
| **版本**     | v2.0                                                                                               |
| **优先级**   | 低                                                                                                 |
| **描述**     | 用户可为重要的版本容器打上自定义标签（如"最终提交版""中期检查版""评审版"），方便快速定位关键版本。 |
| **输入**     | 标签文本                                                                                           |
| **处理**     | 将标签关联到指定版本容器                                                                           |
| **输出**     | 带有自定义标签的版本                                                                               |
| **约束**     | 同一版本可有多个标签，标签支持按标签筛选                                                           |

## 四、非功能需求

### 4.1 性能需求

| 编号 | 需求         | 指标                    |
| ---- | ------------ | ----------------------- |
| NFR1 | 响应速度     | 页面加载时间不超过3秒   |
| NFR2 | 文档编辑响应 | 输入延迟不超过100ms     |
| NFR3 | 存储容量     | 支持至少200MB总存储空间 |

### 4.2 可用性需求

| 编号 | 需求       | 指标                                   |
| ---- | ---------- | -------------------------------------- |
| NFR4 | 易用性     | 新用户无需培训可在10分钟内完成基本操作 |
| NFR5 | 界面一致性 | 所有页面使用统一的UI组件库             |

### 4.3 数据安全

| 编号 | 需求       | 说明                                   |
| ---- | ---------- | -------------------------------------- |
| NFR6 | 数据本地化 | 所有数据存储在浏览器本地，不上传服务器 |
| NFR7 | 数据导出   | 支持将数据导出为JSON文件备份           |

### 4.4 兼容性

| 编号 | 需求       | 说明                           |
| ---- | ---------- | ------------------------------ |
| NFR8 | 浏览器兼容 | 支持Chrome、Edge最新两个主版本 |
