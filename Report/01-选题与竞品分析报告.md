# 选题与竞品分析报告

## 一、选题来源与缘由

### 1.1 选题背景

在研究生课程学习过程中，小组项目是常见的教学形式。然而在实际协作中，普遍存在以下痛点：

1. **版本混乱**：小组成员通过微信群、QQ群互传文件，导致"最终版""最终版2""真正最终版"等文件满天飞，版本管理形同虚设。
2. **修改不可追溯**：谁在什么时候改了什么内容，无从查证。出现问题后难以定位责任人和变更原因。
3. **多类型产出物分散**：一个项目通常包含文档（Word/PPT）、代码、图片、视频等多种类型文件，分散在不同的聊天记录和网盘链接中，缺乏统一管理。
4. **无法协作编辑**：传统方式下同一文档只能一人编辑，其他人等待，效率低下。

### 1.2 选题缘由

基于上述痛点，本选题拟开发一个**轻量级多人文档协作系统**，以"版本容器"为核心概念，将项目的所有产出物（文档、代码、资源）统一纳入一个容器进行版本管理，支持多人实时协作编辑，强制要求每次修改附带标签和描述，实现变更的完整可追溯。

## 二、竞品分析

### 2.1 竞品选择

选取两类目前学生群体中最常用的文件共享与协作方式进行对比分析：

- **竞品A**：微信群/QQ群文件传输
- **竞品B**：网盘（百度网盘/OneDrive）

### 2.2 对比分析

| 对比维度     | 微信群/QQ群文件传输               | 网盘（百度网盘/OneDrive）                         | 本系统（DocCollab）                                       |
| ------------ | --------------------------------- | ------------------------------------------------- | --------------------------------------------------------- |
| **统一管理** | ❌ 文件分散在聊天记录中，难以检索 | ⚠️ 可集中存储，但无项目维度的组织                 | ✅ 以版本容器为单位统一管理所有产出物                     |
| **版本管理** | ❌ 无版本概念，靠文件名区分       | ⚠️ 仅支持简单的历史版本（如OneDrive），无版本标签 | ✅ 完整的版本容器管理，支持版本标签、变更记录             |
| **修改说明** | ❌ 无强制修改说明                 | ❌ 无修改说明机制                                 | ✅ 每次修改必须填写标签（需求变更/Bug修复等）和描述       |
| **文档协作** | ❌ 不支持协作编辑                 | ⚠️ 仅Office Online支持有限的协作                  | ✅ 内置富文本编辑器，支持多人实时协作                     |
| **变更追溯** | ❌ 无法追溯变更历史               | ❌ 仅能查看文件修改时间                           | ✅ 完整的变更记录，可查看每次修改的标签、描述、时间、作者 |

### 2.3 功能缺陷分析

#### 微信群/QQ群文件传输的缺陷

- 文件有过期时间，群文件随时间推移自动清理
- 无任何版本管理能力
- 搜索困难，翻聊天记录如大海捞针
- 不支持多人同时编辑文档
- 无法对修改进行标注和说明

#### 网盘（百度网盘/OneDrive）的缺陷

- 没有"项目"维度的组织概念，所有文件平铺
- 没有强制修改说明，无法了解变更意图
- 缺乏结构化的变更追溯能力
- 百度网盘无在线协作编辑功能
- OneDrive 协作功能受限于 Office 格式文档

## 三、创新点

1. **版本容器**：以"容器"为粒度管理一次完整的项目状态，将文档、代码、资源等所有产出物统一打包，而非零散管理单个文件。
2. **强制修改标签**：每次保存新版本必须选择版本标签（如"初始版本""功能迭代""Bug修复""文档完善""代码重构"）并填写说明，确保每个版本都有明确的创建目的。
3. **轻量级协作**：面向课程小组场景设计，无需复杂部署，浏览器打开即用。采用 BroadcastChannel API 实现同浏览器多标签页实时协作，基于 LocalStorage 本地存储，零运维成本。

## 四、拟定解决的关键问题

| 序号 | 关键问题         | 解决方案                                                 |
| ---- | ---------------- | -------------------------------------------------------- |
| 1    | 版本管理混乱     | 通过版本容器机制，每次保存形成一个快照版本，可随时回溯   |
| 2    | 变更无法追溯     | 强制修改标签+描述，形成完整的变更记录链                  |
| 3    | 多类型产出物分散 | 将所有文件统一纳入容器管理，支持文档、代码、图片、视频等 |
| 4    | 协作编辑效率低   | 基于富文本编辑器实现多人实时协作编辑文档                 |

---

## 五、小结

本系统定位于面向课程小组的轻量级协作工具，与微信群文件传输和传统网盘相比，在版本管理、变更追溯、协作编辑三个维度具有显著优势。系统以"版本容器"为核心创新点，通过强制修改标签机制确保变更可追溯，符合CMMI配置管理和需求管理的思想理念。
