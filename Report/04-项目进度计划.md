## 一、引言

### 1.1 目的

本文档定义"多人文档协作系统（DocCollab）"项目的进度安排，通过工作分解结构（WBS）和甘特图明确各阶段的任务分工和时间节点，确保项目按时交付。

### 1.2 项目周期

- **总周期**：4周（28天）
- **起止日期**：2026年5月18日 — 2026年6月14日

## 五、风险计划

### 5.1 风险识别

| 风险编号 | 风险描述                       | 概率 | 影响 | 等级  |
| -------- | ------------------------------ | ---- | ---- | ----- |
| RSK-01   | 技术选型不熟悉，学习成本高     | 中   | 中   | ⚠️ 中 |
| RSK-02   | 单人开发，时间紧张             | 高   | 高   | 🔴 高 |
| RSK-03   | 富文本编辑器集成遇到兼容性问题 | 中   | 中   | ⚠️ 中 |
| RSK-04   | 浏览器存储容量限制导致功能受限 | 低   | 中   | 🟡 低 |
| RSK-05   | 需求理解偏差导致返工           | 低   | 高   | ⚠️ 中 |

### 5.2 风险应对策略

| 风险编号 | 应对策略 | 具体措施                                                             |
| -------- | -------- | -------------------------------------------------------------------- |
| RSK-01   | 缓解     | 优先选择文档完善、社区活跃的技术方案（Vue3 + Element Plus + TipTap） |
| RSK-02   | 缓解     | 严格按计划执行，简化非核心功能；单人承担多角色                       |
| RSK-03   | 缓解     | 选用成熟的 TipTap 编辑器，避免从零开发                               |
| RSK-04   | 接受     | 明确告知用户存储限制，提供 ZIP 导出备份功能                          |
| RSK-05   | 避免     | 开发前充分确认需求文档，与课程要求对齐                               |
